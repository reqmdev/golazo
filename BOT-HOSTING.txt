Bot / Pterodactyl hosting setup (Bot-Hosting.net, Wispbyte, etc.)
==================================================================

ERROR: Cannot find module '/home/container/src/index.js' (Wispbyte)
→ The panel looks for that exact path. Your files may be inside /home/container/Golazo/.
  Pick ONE fix:
  A) Move everything from Golazo/ up to /home/container/ (Files → select all → Move → ..)
  B) Set JS file to: Golazo/src/index.js  (if you keep the subfolder)
  C) Set JS file to: Golazo/index.js  (works after npm install — recommended if stuck in Golazo/)

Wispbyte / picky panels
-----------------------
- JS file: src/index.js  (when package.json is in /home/container)
- JS file: Golazo/src/index.js  (when the GitHub zip left a Golazo folder)
- Golazo v1.0+ auto-detects package.json and fixes working directory — no manual cd needed
- After upload: Stop → delete node_modules → Start once (let npm install finish)

ERROR: Cannot find module '/home/container/index.js'
→ Your files are not in the right place OR the startup file is wrong.

ERROR: SyntaxError in node_modules/typescript/lib/_tsc.js (Unexpected end of input)
→ npm install was interrupted OR node_modules is corrupted. The repo ships pre-built
  dist/graphics so TypeScript does not need to compile on the server.

FIX — Startup tab (most important)
----------------------------------
1. Open your server → Startup tab
2. Set the main command to (NOT "npm start"):
   cd /home/container && if [ -f package.json ]; then npm install --no-fund --no-audit; fi && node .
3. If there is a "Bot Node.js file" field, set it to: index.js
4. Save → Stop → delete the node_modules folder (Files tab) → Start once

Why not "npm start"?
- npm start runs nodemon (dev dependency) and tries to rebuild graphics on every start.
- node . runs the bot directly (production mode).

FIX — Files in a subfolder
--------------------------
If you uploaded a zip, files may be in /home/container/Golazo/ instead of /home/container/
1. Files tab → open the Golazo folder
2. Select all files → Move → enter: ..
3. Delete empty folder and zip
4. Restart

FIX — Fresh upload from GitHub
------------------------------
1. Delete all files in /home/container (or create new server)
2. Upload latest zip from https://github.com/reqmdev/golazo (Code → Download ZIP)
3. Unarchive → move files to /home/container root (see above)
4. Startup command:
   cd /home/container && if [ -f package.json ]; then npm install --no-fund --no-audit; fi && node .
5. Settings/Variables → add CLIENT_TOKEN, MONGODB_URI, GOLAZO_OWNER_ID
6. Start

FIX — Corrupted node_modules (TypeScript / build errors)
--------------------------------------------------------
1. Stop the server (do not restart in a loop)
2. Files tab → delete the entire node_modules folder
3. Confirm dist/graphics/adapters/index.js exists in your files (comes with the GitHub zip)
4. Start once and wait for npm install to finish completely before touching Restart

Required environment variables
------------------------------
CLIENT_TOKEN=
MONGODB_URI=mongodb+srv://...@cluster.mongodb.net/golazo?retryWrites=true&w=majority
GOLAZO_OWNER_ID=your_discord_user_id
GOLAZO_ENABLE_DEV_COMMANDS=false
GOLAZO_DEV_MODE=false
